# RM6020 开环测试说明

## 1. 目标
本测试基于当前 `F:\RM\test` 工程，验证 C 板框架下 `GM6020` 电机的 CAN2 通信、任务调度和基础开环输出是否正常。

本测试为开环测试：
- 不使用 PID
- 不做速度环或位置环
- 周期发送固定输出值

## 2. 参考配置
引脚和 CAN 配置优先参考：

`E:\honor\RM\未命名文件夹(1)\未命名文件夹`

当前工程按 C 板口径使用：
- `CAN2_RX = PB5`
- `CAN2_TX = PB6`
- `APB1 = 42 MHz`
- `CAN2 = 1 Mbps`
- `Prescaler = 3`
- `BS1 = 12TQ`
- `BS2 = 1TQ`

## 3. 已修改文件

- `Src/freertos.c`
  增加 `Task_Init` 创建
- `User/task/init.c`
  在初始化任务中调用 `BSP_CAN_Init()`，再创建 `Task_RM6020`
- `User/task/6020.c`
  实现 `RM6020` 开环测试任务
- `Src/can.c`
  将 `CAN2` 时序改为参考工程的 `3/12/1`

## 4. 默认测试参数
当前默认参数位于 `User/task/6020.c`：

```c
#define RM6020_TEST_MOTOR_ID         0x205u
#define RM6020_TEST_OUTPUT           0.05f
#define RM6020_TEST_START_DELAY_MS   1000u
```

含义如下：
- 电机反馈 ID 默认使用 `0x205`
- 上电后延时 `1000 ms` 再开始输出
- 开环输出固定为 `0.05`，范围是 `[-1.0, 1.0]`

如果你的电机不是 `0x205`，把 `RM6020_TEST_MOTOR_ID` 改成实际 ID，例如 `0x206`。

## 5. 运行流程
1. `main.c` 初始化 `GPIO` 和 `CAN2`
2. `MX_FREERTOS_Init()` 创建 `Task_Init`
3. `Task_Init()` 调用 `BSP_CAN_Init()`
4. `Task_Init()` 创建 `Task_RM6020`
5. `Task_RM6020()` 注册电机并以 `500 Hz` 周期发送固定输出

## 6. 测试步骤
1. 确认电机接在 `CAN2(PB5/PB6)`，总线供电和收发器正常
2. 确认电机机械上安全，首次建议悬空测试
3. 编译并下载程序
4. 上电后等待约 `1 s`
5. 观察电机是否以小输出缓慢转动

## 7. 安全建议
- 首次测试不要把 `RM6020_TEST_OUTPUT` 调得过大，建议先保持 `0.05`
- 如果方向相反，可改：

```c
.reverse = true
```

- 如果需要反向测试，也可以把 `RM6020_TEST_OUTPUT` 改成负值
- 出现抖动、堵转、异常发热时，立刻将输出改为 `0.0f`
